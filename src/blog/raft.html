what this project is

kill-my-cluster is a custom raft kv written in go. it runs seven dockerized machines with a whitelist control plane for kill, partition, and heal.

grafana and prometheus sit under the demo so you can see leader, term, commit index, and applies move on real processes. the teaching beat is killing the leader and watching the group elect and catch up.

learn raft

if raft is new, start with the secret lives of data. it walks through election and log replication with an interactive timeline.

then read in search of an understandable consensus algorithm. the hub at raft.github.io collects talks and links.

learn grpc

the nodes talk with grpc over http/2, using protocol buffers for the messages.

good starting docs: core concepts and the go quickstart.

how the repo maps

internal/storage is the wal and snapshots. internal/raft is election and replication. internal/kv is the replicated map with exactly-once client ids.

internal/controlplane is whitelist kill, partition, heal, and sse. deploy/compose runs the seven nodes plus prometheus and grafana. locally that is control plane :8080, grafana :3001, and prometheus :9090. the public ui is this page, not a separate demo host.

suggested path

  1. skim the dashboard above so the controls are familiar.
  2. watch the raft talk, then skim the paper.
  3. watch grpc in five minutes.
  4. clone kill-my-cluster and run compose when you want the real cluster.
  5. read internal/raft/node.go with the paper open beside it.
